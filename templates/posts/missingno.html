Background

Once your data is safely localized, one of the first and most important things that you have to do at the beginning of any data analytics project is taking "a lay of the land" with your data. Data is fundamentally messy: full of oddities and noise and incomplete entries. Getting a handle on this weirdness is an essential first step to getting anything actually done with it, and as much as 80% of project time might end up getting sunk in it.

When I got to work with the motor collision dataset that consumed all of my time last weekend I was surprised to find the extent to which missing entries populated the picture. After digging around a bit I discovered that though Python has world-beating facilities for handling missing data, there isn't much out there in terms of visual summary (no, pd.describe() doesn't count!).

I was looking in particular for some way of constructing a missing data matrix: take a sample of some thousand or so entries in your dataset and throw it at the wall to see which columns stick and which ones don't. Surprisingly, as far as I can tell (and do inform me if I'm wrong, reader) there is nothing optimized for this task off-the-shelf. I mean, it's not hard to whip out a crude seaborn.heatmap, but the result is distinctly visually wanting.

So build one, I said! And like any good developer on a hunch I ignored all that other life stuff to punch one out.

Results

The missingno package on PyPI (still can't believe that package name was never taken) exposes the top-level missingno function, which takes a pandas DataFrame as an input and returns an up-tweaked data nullity matrix visualization, built with matplotlib, in return.

For the purposes of demonstration, let's download and visualize 100 random records in the New York City Motor Vehicle Collisions dataset. Once you have the data localized in a DataFrame, all you have to do is: 

    
        >>> from missingno import missingno
        >>> missingno(df.sample(100))

Which results in:

The grays are filled values while the whites are missing data, while the sparkline on the right-hand side visually summarizes data completeness (here you can see that data completeness shelves fairly strongly against 23 values filled). missingno will summarize whatever size DataFrame you throw at it—here is what it looks like with 1000 samples:

missingno takes optional arguments controlling figure size, width ratio, color, and font size, all of whcih are documented over at the GitHub repository. Head over there to learn more, or pip install missingno right away to start using it right away.

If you like this project, be sure to also check out Jos Polfliet's fantastic pandas profiling tool.
