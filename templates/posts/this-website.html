I've been kicking around the idea of creating my own website for some time now. I started learning how to use Python's Flask web framework early this summer, while working on a few publication scripts for the Wikipedia Signpost. The application I was hoping to build and deploy on Wikimedia Labs still isn't quite finished: converting a bunch of scripts meant to be run by hand into a series of scheduled jobs on a server somewhere is far harder to dive into than I thought it would be. Still, that project progresses, albeit slowly (watch this space).

In the meantime I put the skills to use building a home on the web for a CUNY Baruch on-campus organization that I'm active in, the Mathematics Society. What I love about the Flask microframework is the lightness: packages exist for handling most or all of the issues you run into in web programming, but they're hovering in the background of what you're doing, and never intrude. I love Squarespace (and their office seems like a dream to work in) but for a first effort I preferred, for my own elucidation, to create something handbuild.

After I finished that project, it seemed only a matter of time until I turned to creating a personal one as well: and now it has arrived! This website, like the one for the Mathematics Society that preceded, is a static (so far) no-fluff handbuilt project developed in Flask-Python. You can see the source code on GitHub. I use PythonAnywhere as my webhost. Math formulae are rendered with MathJax. iPython notebooks I host on GitHub and render via Jupyter. Blog commentary is powered by Disqus.

So there you go: from zero to semi-reasonable hero in a few months' hacking. Here's to years more of the same to come.
